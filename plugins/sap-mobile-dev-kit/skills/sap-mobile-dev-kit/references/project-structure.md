# MDK Project Structure Reference

## Project Layout

```
MyWarehouseApp.mdkproject/
  ├── Application.app           — Entry point: services, offline config, main page
  ├── Metadata.json             — BAS metadata (app ID, version, Mobile Services config)
  ├── .project.json             — BAS project config (do not hand-edit)
  │
  ├── Pages/                    — Page definition JSON files (.page)
  │     ├── Main.page
  │     ├── Delivery/
  │     │     ├── DeliveryList.page
  │     │     └── DeliveryDetail.page
  │     └── Task/
  │           ├── TaskList.page
  │           └── TaskConfirm.page
  │
  ├── Actions/                  — Action definition JSON files (.action)
  │     ├── Delivery/
  │     │     ├── NavToDeliveryDetail.action
  │     │     └── PostGoodsIssue.action
  │     └── Common/
  │           ├── ShowError.action
  │           └── NavBack.action
  │
  ├── Rules/                    — JavaScript rule files (.js)
  │     ├── Delivery/
  │     │     ├── GetDeliveryFilter.js
  │     │     └── ValidateDelivery.js
  │     └── Task/
  │           └── ConfirmTask.js
  │
  ├── Services/                 — Service definition files (.service)
  │     └── EWM_SERVICE.service
  │
  ├── i18n/                     — Internationalization
  │     ├── i18n.properties         (default — English)
  │     ├── i18n_de.properties      (German)
  │     └── i18n_fr.properties      (French)
  │
  └── Images/                   — App images
        ├── logo.png
        └── warehouse_icon.png
```

## Application.app

```json
{
  "Application": {
    "Name": "EWM Warehouse",
    "Version": "1.0.0",
    "MainPage": "/MyWarehouseApp/Pages/Main.page",
    "OnDidUpdate": "/MyWarehouseApp/Actions/SyncOfflineStore.action",
    "Services": [],
    "OfflineEnabled": true,
    "AppId": "com.mycompany.ewm.warehouse"
  }
}
```

## Service Definition (.service)

```json
{
  "_Type": "MDK.ODataService",
  "_Name": "EWM_SERVICE",
  "ServiceUrl": "/sap/opu/odata/sap/ZEWM_DELIVERY_SRV/",
  "Destination": "EWM_BACKEND",
  "ODataVersion": "V2",
  "Offline": true
}
```

`Destination` must match a destination configured in BTP destination service and linked in Mobile Services.

## Metadata.json

Generated by BAS — do not hand-edit critical fields:

```json
{
  "Application": {
    "BundleId": "com.mycompany.ewm.warehouse",
    "MobileServicesAppId": "com.mycompany.ewm.warehouse",
    "Version": "1.0.0",
    "MinimumMDKVersion": "23.4"
  }
}
```

## Naming Conventions

| Type | Convention | Example |
|---|---|---|
| Pages | PascalCase noun | `DeliveryDetail.page` |
| Actions | Verb + noun | `NavToDeliveryDetail.action`, `PostGoodsIssue.action` |
| Rules | Verb + noun | `ValidateQuantity.js`, `GetTaskFilter.js` |
| Services | SCREAMING_SNAKE | `EWM_SERVICE.service` |
| i18n keys | UPPER_SNAKE | `DELIVERY_LIST_TITLE` |

## BAS MDK Editor Setup

```
1. Open Business Application Studio
2. Create Dev Space with "SAP Mobile Application" extension
3. File → New Project → MDK Project
   Template: List-Detail App or Empty App
   App ID: com.mycompany.ewm.warehouse
   Destination: EWM_BACKEND (pre-configured in BTP)
   Service: /sap/opu/odata/sap/ZEWM_DELIVERY_SRV/

4. MDK Explorer panel appears on left (page tree, action tree, rule tree)
5. Click any .page → visual editor opens
6. Switch to code view (</> button) for JSON editing
```

## Adding a New Page

```
BAS: Right-click Pages folder → MDK: New Page
  Template: ListReport / SectionedTable / FormCell / ObjectTable
  Name: DeliveryList

" Or create .page file manually and paste JSON template
```

## Referencing Files in JSON

All file references use absolute paths from project root:

```json
"/MyWarehouseApp/Pages/DeliveryDetail.page"
"/MyWarehouseApp/Actions/PostGoodsIssue.action"
"/MyWarehouseApp/Rules/ValidateQty.js"
"/MyWarehouseApp/Services/EWM_SERVICE.service"
```
